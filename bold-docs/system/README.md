# System Taxonomy

Product Owner TL;DR: These docs describe the current intended RITE system. They are the durable reference for future planning and implementation, separate from the prototype files that inspired them.

## Docs

- `product.md` - product scope, audience, content areas, and roadmap.
- `architecture.md` - current technical direction for API, frontend, storage, hosting, and authentication.
- `privacy-and-governance.md` - roles, permissions, review workflow, and privacy rules.
- `design-system.md` - West Wichita Rotary brand and UI conventions discovered from the prototype design system.
- `prototype.md` - how to use the feature-local prototype as reference material without treating it as final site source.
